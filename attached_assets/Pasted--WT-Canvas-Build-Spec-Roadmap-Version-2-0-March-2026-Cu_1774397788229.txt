**Version 2.0 | March 2026**

## Current State Summary

A block-based content management system for blog articles, landing pages, and lead magnets. AI-assisted via OpenAI GPT-4o. Content stored in Supabase. Auth in Replit PostgreSQL. Media via Cloudinary.

## Phase 0 — Foundation Fixes (Do First, ~2–3 days)

These changes should happen before anything else. They unlock everything below.

### 0A — Consolidate to Supabase Auth

**Remove:** Replit PostgreSQL entirely.
**Replace with:** Supabase Auth (built-in).

Steps:
1. Enable Supabase Auth in your Supabase project dashboard
2. Create a `profiles` table in Supabase linked to `auth.users` for storing team member display names, roles, etc.
3. Replace Express session middleware + bcrypt with Supabase Auth client-side login (`supabase.auth.signInWithPassword`)
4. Replace JWT middleware on the backend with Supabase JWT verification (`supabase.auth.getUser(token)`)
5. Update RLS (Row Level Security) policies on all Supabase tables to scope by `auth.uid()`
6. Remove the Replit PostgreSQL environment variable and all `pg` / `drizzle` (Replit DB) imports

**Result:** Single database, cleaner auth, proper RLS, and the `sessions` table disappears entirely.

## Phase 1 — SEO Page Delivery (Cloudflare Worker Architecture)

### Architecture Overview

Two completely separate delivery systems that share Supabase as the content store:

```
┌─────────────────────────────────────────────────────┐
│  CAPABILITY 1: SEO Web Pages                        │
│                                                     │
│  Canvas (Replit) ──writes──▶ Supabase               │
│                                   │                 │
│  Google / User ──▶ Cloudflare Worker ──reads──▶ Supabase
│                       │                             │
│                    renders HTML                     │
│                    caches at edge                   │
│                    serves globally                  │
│                                                     │
│  Replit is NEVER in the public serving path         │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  CAPABILITY 2: Email / SMS Campaigns                │
│                                                     │
│  Canvas (Replit) ──builds──▶ renders HTML           │
│                                   │                 │
│                              pushes to Klaviyo      │
│                              Klaviyo sends it       │
│                                                     │
│  Cloudflare is NOT involved here                    │
└─────────────────────────────────────────────────────┘
```

These two capabilities share: block editor, Claude AI, Shopify connection, Supabase database, and media library. They diverge completely at point of delivery.

### 1B — Block Renderer (Shared Module)

This renderer is written once and used in two places: the Canvas preview (Replit) and the Cloudflare Worker. Keep it in a standalone file with zero Node.js dependencies so it runs in both environments.

Create `/src/renderer/blockToHtml.ts`:

```typescript
// Zero external dependencies — must run in both Node.js (Replit) and Workers (Cloudflare)
export function renderBlock(block: ContentBlock): string {
  switch (block.type) {
    case 'heading':
      return `<h${block.level} class="wt-heading wt-h${block.level}">${escHtml(block.text)}</h${block.level}>`;
    case 'paragraph':
      return `<p class="wt-paragraph">${escHtml(block.text)}</p>`;
    case 'image':
      return `<figure class="wt-figure">
        <img src="${escAttr(block.url)}" alt="${escAttr(block.alt)}" loading="lazy" />
        ${block.caption ? `<figcaption>${escHtml(block.caption)}</figcaption>` : ''}
      </figure>`;
    case 'quote':
      return `<blockquote class="wt-quote">${escHtml(block.text)}</blockquote>`;
    case 'list':
      const tag = block.ordered ? 'ol' : 'ul';
      return `<${tag} class="wt-list">${block.items.map(i => `<li>${escHtml(i)}</li>`).join('')}</${tag}>`;
    case 'cta':
      return `<div class="wt-cta"><a href="${escAttr(block.url)}" class="wt-cta-button">${escHtml(block.text)}</a></div>`;
    case 'html_block':
      return block.html; // raw HTML — only used by trusted internal authors
    default:
      return '';
  }
}

export function renderPageHtml(page: Page, baseUrl: string): string {
  const body = page.content_json.blocks.map(renderBlock).join('\n');
  const canonical = page.canonical_url || `${baseUrl}/${page.slug}`;
  const ogTitle = page.og_title || page.title;
  const ogImage = page.og_image || page.featured_image || '';

  // JSON-LD structured data
  const schema = page.structured_data || buildDefaultSchema(page, baseUrl);

  return `<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>${escHtml(page.title)}</title>
  <meta name="description" content="${escAttr(page.meta_description || '')}" />
  <link rel="canonical" href="${escAttr(canonical)}" />

  <!-- Open Graph -->
  <meta property="og:type" content="article" />
  <meta property="og:title" content="${escAttr(ogTitle)}" />
  <meta property="og:description" content="${escAttr(page.meta_description || '')}" />
  <meta property="og:url" content="${escAttr(canonical)}" />
  ${ogImage ? `<meta property="og:image" content="${escAttr(ogImage)}" />` : ''}
  <meta property="og:site_name" content="Well Told" />

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image" />
  <meta name="twitter:title" content="${escAttr(ogTitle)}" />
  <meta name="twitter:description" content="${escAttr(page.meta_description || '')}" />
  ${ogImage ? `<meta name="twitter:image" content="${escAttr(ogImage)}" />` : ''}

  <!-- JSON-LD Structured Data -->
  <script type="application/ld+json">${JSON.stringify(schema)}</script>

  <!-- Page styles -->
  <link rel="stylesheet" href="${baseUrl}/styles/wt-pages.css" />
  ${page.custom_css ? `<style>${page.custom_css}</style>` : ''}
</head>
<body class="wt-page wt-template-${page.page_template || 'default'}">
  <main class="wt-content">
    ${body}
  </main>
</body>
</html>`;
}

// Default Article schema — override with page.structured_data for custom types
function buildDefaultSchema(page: Page, baseUrl: string) {
  return {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": page.title,
    "description": page.meta_description,
    "url": `${baseUrl}/${page.slug}`,
    "datePublished": page.published_at,
    "dateModified": page.updated_at,
    "image": page.og_image || page.featured_image,
    "publisher": {
      "@type": "Organization",
      "name": "Well Told",
      "url": baseUrl
    }
  };
}

function escHtml(str: string) {
  return str.replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;');
}
function escAttr(str: string) {
  return str.replace(/"/g,'&quot;').replace(/'/g,'&#39;');
}
```

### 1C — Cloudflare Worker

Create a new Cloudflare Worker project (separate from Replit). This is a small standalone deployment — approximately 150 lines total.

**Setup:**
```bash
npm create cloudflare@latest wt-pages-worker
cd wt-pages-worker
npm install @supabase/supabase-js
```

**Worker environment variables** (set in Cloudflare dashboard):
```
SUPABASE_URL=
SUPABASE_ANON_KEY=      # read-only, public — safe here
SITE_BASE_URL=https://yourdomain.com
```

**`src/index.ts`:**
```typescript
import { createClient } from '@supabase/supabase-js';
import { renderPageHtml } from './renderer/blockToHtml'; // shared module, copied here

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    const url = new URL(request.url);
    const path = url.pathname;

    // Sitemap
    if (path === '/sitemap.xml') return handleSitemap(env);

    // Robots
    if (path === '/robots.txt') return handleRobots(env);

    // Redirect table — check for old URLs that now 301 elsewhere
    const redirect = await checkRedirects(path, env);
    if (redirect) return Response.redirect(redirect, 301);

    // Page render
    return handlePage(path, env);
  }
};

async function handlePage(path: string, env: Env): Promise<Response> {
  const slug = path.replace(/^\//, '').replace(/\/$/, '') || 'home';
  const supabase = createClient(env.SUPABASE_URL, env.SUPABASE_ANON_KEY);

  // Try blog articles first, then landing pages
  const sources = ['blog_articles', 'landing_pages', 'lead_magnets'];
  let page = null;

  for (const table of sources) {
    const { data } = await supabase
      .from(table)
      .select('*')
      .eq('slug', slug)
      .eq('status', 'live')
      .single();
    if (data) { page = data; break; }
  }

  if (!page) {
    return new Response(render404(), {
      status: 404,
      headers: { 'Content-Type': 'text/html' }
    });
  }

  const html = renderPageHtml(page, env.SITE_BASE_URL);

  return new Response(html, {
    status: 200,
    headers: {
      'Content-Type': 'text/html; charset=utf-8',
      // Cache at Cloudflare edge for 1 hour; revalidate in background
      // Canvas publish flow purges this cache immediately on publish
      'Cache-Control': 'public, max-age=3600, stale-while-revalidate=86400',
      'X-Robots-Tag': 'index, follow',
    }
  });
}

async function handleSitemap(env: Env): Promise<Response> {
  const supabase = createClient(env.SUPABASE_URL, env.SUPABASE_ANON_KEY);
  const base = env.SITE_BASE_URL;

  const [blogs, pages] = await Promise.all([
    supabase.from('blog_articles').select('slug, updated_at').eq('status', 'live'),
    supabase.from('landing_pages').select('slug, updated_at').eq('status', 'live'),
  ]);

  const urls = [
    ...(blogs.data || []),
    ...(pages.data || []),
  ].map(p => `
  <url>
    <loc>${base}/${p.slug}</loc>
    <lastmod>${p.updated_at?.split('T')[0]}</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>`).join('');

  const xml = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${urls}
</urlset>`;

  return new Response(xml, {
    headers: {
      'Content-Type': 'application/xml',
      'Cache-Control': 'public, max-age=3600',
    }
  });
}

async function handleRobots(env: Env): Promise<Response> {
  const body = `User-agent: *
Allow: /
Sitemap: ${env.SITE_BASE_URL}/sitemap.xml

# Block admin/editor paths
Disallow: /api/
Disallow: /admin/`;

  return new Response(body, {
    headers: { 'Content-Type': 'text/plain' }
  });
}

async function checkRedirects(path: string, env: Env): Promise<string | null> {
  // Query Supabase for any page that has this path in redirect_from[]
  const supabase = createClient(env.SUPABASE_URL, env.SUPABASE_ANON_KEY);
  const slug = path.replace(/^\//, '');
  const { data } = await supabase
    .from('blog_articles')
    .select('slug')
    .contains('redirect_from', [slug])
    .single();
  return data ? `${env.SITE_BASE_URL}/${data.slug}` : null;
}
```

### 1D — Canvas: Publish Flow

When a user clicks "Publish" in Canvas, the backend does three things:

```typescript
// /server/routes/publish.ts
app.post('/api/publish/:table/:id', async (req, res) => {
  const { table, id } = req.params;

  // 1. Set status = 'live' and published_at in Supabase
  await supabase
    .from(table)
    .update({ status: 'live', published_at: new Date().toISOString() })
    .eq('id', id);

  // 2. Purge Cloudflare cache for this slug so next request gets fresh render
  const { data: page } = await supabase.from(table).select('slug').eq('id', id).single();
  await purgeCloudflareCache(`${process.env.SITE_BASE_URL}/${page.slug}`);

  // 3. Ping Google Search Console to re-crawl (optional but useful)
  // fetch(`https://www.google.com/ping?sitemap=${process.env.SITE_BASE_URL}/sitemap.xml`)

  res.json({ success: true, url: `${process.env.SITE_BASE_URL}/${page.slug}` });
});

async function purgeCloudflareCache(url: string) {
  await fetch(
    `https://api.cloudflare.com/client/v4/zones/${process.env.CF_ZONE_ID}/purge_cache`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.CF_API_TOKEN}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ files: [url] }),
    }
  );
}
```

The page builder in Canvas uses a hybrid pattern: template picker first, then block editor.

**Step 1 — Template picker (modal on "New Page")**

type PageLayoutTemplate = {

**SEO fields panel** (right sidebar in page editor):
- Slug (auto-generated from title, editable)
- Meta description (with AI generate button + character counter)
- Primary keyword + supporting keywords (existing fields)
- Featured image / OG image (media picker)
- OG title (defaults to page title, override available)
- Canonical URL (defaults to `yourdomain.com/slug`, override for syndicated content)

- Redirect from (add old URLs that should 301 to this page)

**Publish button states:**
- `Draft` → "Save Draft"
- `Review` → "Submit for Review"
- `Approved` → "Schedule" or "Publish Now"

### 2B — Campaign Type & Data Resolution Architecture

This is a first-class architectural decision that affects how every Shopify-connected and dynamic block behaves.

#### The two modes

**Campaigns** (2–3x/week sends, holiday blitzes) use **send-time snapshot**:
- All Shopify product data — images, prices, URLs, inventory state — is resolved and frozen when the campaign is scheduled or sent
- The email always shows the price and product you intentionally curated, regardless of when it is opened
- Protects promotional integrity: a flash sale email opened after the sale still shows the sale offer
- Stored in `snapshot_json` on the campaign record

**Flows** (evergreen behavioral triggers — abandoned cart, win-back, post-purchase) use **open-time resolution**:
- Shopify-connected blocks fetch live data via a dynamic image service or URL template at the moment the email is opened
- Prices, inventory, and images always reflect current reality
- Prevents broken experiences: an abandoned cart email opened 48 hours later won't link to a sold-out product

#### Campaign-level toggle

Set automatically based on `campaign_type`, overridable by the editor:

```typescript
type EmailCampaign = {
  campaign_type: 'campaign' | 'flow' | 'transactional'
  shopify_resolution: 'send_time' | 'open_time'
  // send_time → snapshot_json populated at schedule time
  // open_time → blocks render with live URL templates at open time
}

// Default resolution by type:
const defaultResolution: Record<string, ShopifyResolution> = {
  campaign:      'send_time',
  flow:          'open_time',
  transactional: 'send_time',
}
```

In the Canvas UI this surfaces as a clearly labelled badge on the campaign card and a toggle in campaign settings — no hidden behaviour.

#### Hybrid pattern for flows: snapshot the offer, live-check inventory

For flows, the recommended approach is not fully live — it is a hybrid:

1. **Copy and creative** → snapshotted at flow build time (your curated headline, offer text, brand assets). These don't change.
2. **Product image and URL** → live-resolved at open time via Shopify Storefront API or dynamic image URL
3. **Inventory check** → run at send time (when Klaviyo triggers the flow send). If the product variant is out of stock, either:
   - Swap to a related product (preferred — keeps the email sending)
   - Suppress the send entirely (Klaviyo flow filter)

This gives you stable creative with live product truth, without the risk of sending someone to a 404 or a sold-out PDP.

```typescript
// Flow email block with hybrid resolution
type ProductCardBlock = {
  type: 'product_card'
  shopify_source: ShopifySource
  resolution: 'open_time'

  // Snapshotted at flow build — editorial, stable
  headline: string
  cta_text: string
  offer_copy?: string

  // Live at open time — Shopify Storefront API
  image_url: string       // → dynamic URL template
  title: string           // → merge tag
  price: string           // → merge tag
  product_url: string     // → merge tag

  // Inventory fallback
  oos_behavior: 'swap_related' | 'suppress' | 'show_anyway'
  fallback_product_id?: string
}
```

## Key Technical Notes for Replit

- Use `tsx` for running TypeScript server files: `"dev": "tsx watch server/index.ts"`
- Supabase client should be initialized once and exported: `/server/lib/supabase.ts`
- All Klaviyo and Shopify Admin API calls must go through the backend (never expose private keys to frontend)
- Shopify Storefront API token is safe to use from the frontend for product search
- Email HTML rendering is server-side on Replit (Klaviyo delivery path)
- SEO page rendering is in the Cloudflare Worker (public delivery path) — Replit never serves public pages
- The block renderer (`blockToHtml.ts`) must have zero Node.js dependencies so it runs in both environments
- Snapshot resolution (`resolveSnapshot`) runs server-side at schedule time — never in the browser
- Use TanStack Query's `invalidateQueries` after every mutation (already partially in place)
- Add Zod schemas for all Klaviyo and Shopify API payloads before sending
- Product cache in Supabase reduces Shopify API calls during heavy editing sessions
